# Results and Metrics

This directory holds the numeric outputs of the evaluation pipeline: the headline metrics, the full per-class and aggregate tables, the reliability and entropy data behind the calibration analysis, and the class composition of each dataset. These files are what the manuscript tables and figures are built from, and they allow every quoted value to be checked independently.

## Directory layout

```
results/
├── bloodmnist/
├── breastmnist/
├── dermamnist/
└── organamnist/
```

## File guide

Each dataset folder contains three CSV files and six JSON files. The block below uses BloodMNIST as the example; the other folders mirror it with their own dataset prefix on the CSV files.

| File | Format | Contents |
| --- | --- | --- |
| `<dataset>_compact_metrics.csv` | CSV | Headline metric set, one row per configuration: accuracy, AUC, macro precision, recall, F1, PR-AUC, expected calibration error (ECE), and log loss. This is the table that maps to the main results table in the manuscript. |
| `<dataset>_summary_metrics.csv` | CSV | The fuller aggregate table, with precision, recall, F1, and PR-AUC reported as both macro and weighted averages, plus macro and weighted AUC, accuracy, and log loss. Weighted recall equals overall accuracy by construction and is listed for completeness. |
| `class_distribution.csv` | CSV | Per-class support counts and percentage share for the dataset, matching the class composition used in the label tables. |
| `ConvNeXtBase_results.json` | JSON | Complete result dump for ConvNeXt-Base (see structure below). |
| `ViT-Base_results.json` | JSON | Complete result dump for ViT-Base. |
| `EfficientNetV2M_results.json` | JSON | Complete result dump for EfficientNetV2-M. |
| `InceptionResNetV2_results.json` | JSON | Complete result dump for InceptionResNetV2. |
| `SoftVoting_results.json` | JSON | Complete result dump for the Soft Voting ensemble. |
| `RigorousStacking_results.json` | JSON | Complete result dump for the Rigorous Stacking ensemble. |

## What the JSON files hold

The per-configuration JSON files are the most detailed artefact in the repository. Each one collects, for a single model or ensemble on a single dataset, the quantities that the compact and summary CSVs condense and that the figures visualise. Typical contents include:

- the scalar discrimination metrics (accuracy, macro and weighted AUC, precision, recall, F1, PR-AUC, log loss);
- per-class precision, recall, F1, and support;
- the confusion matrix;
- the descriptive statistics of the Shannon-entropy distributions, split by correct and incorrect predictions, together with the kernel-density peak locations and heights;
- the bin-level reliability data: per-bin mean confidence, empirical accuracy, gap, and sample count, from which ECE is computed.

Because the bin construction follows each model's own reliability curve, the number of populated bins can vary between models, and per-bin counts are not constrained to sum to the test-set total.

## How the files relate

- The CSV files are convenient summaries for quick comparison across configurations.
- The JSON files are the source of truth: the CSVs, the manuscript tables, and the figure panels are all derived from them.
- `class_distribution.csv` documents the class ordering used throughout, which is the same ordering as the integer labels in `predictions/<dataset>/y_true_flat.npy`.

## Reproducing these files

The results are regenerated by `notebooks/<dataset>/01_<dataset>_ensemble_study_evaluation_analysis.ipynb`, which reads the arrays in `predictions/<dataset>/` and writes the CSV and JSON outputs here. Because that path operates on the saved predictions, it does not require a GPU.

These numbers describe model behaviour on standardised public benchmark splits. They are released for verification and are not clinical performance claims.
